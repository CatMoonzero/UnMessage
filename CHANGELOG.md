# Changelog

本项目的所有重大变更都将记录在此文件中。

## [1.1.0] - 2026-04-01

> 大版本更新：多群聊并发、群生命周期完善、代码健壮性与文档全面整理。

### Added

- **多群聊并发支持**：
  - 用户可同时加入多个群聊，每个群聊独立密钥、独立会话标签页
  - 服务端以 `JoinedGroups` 字典替代单一 `GroupId` 字段，跟踪每个用户的多群成员关系
  - 客户端群密钥、密钥交换、密钥持有者状态均按群 ID 独立维护
- **右键菜单“退出群聊”按钮**：
  - 管理员退出时自动移交管理权给群内其他成员
  - 普通成员退出仅清理本地群会话
- **管理员身份提示**：
  - 创建群聊者、管理员移交接任者、空群首位加入者均会收到系统提示“你已成为群聊管理员”
- **在线列表快捷回会话能力**：
  - 当与某在线用户已存在会话时，点击“在线”列表中的该用户可直接返回该会话

### Changed

- **结束聊天语义调整**：
  - 对群聊点击“结束聊天”只关闭标签页，不触发退出群聊操作
  - 关闭后可双击群列表重新打开（不需重新加入）
  - 对私聊行为不变
- **协议负载升级**：
  - 所有群聊协议负载均增加 `GroupId` 字段（`GroupChatMessage`、`GroupKeyExchangeRequest`、`GroupKeyExchangeForward`、`GroupKeyDeliverRequest`、`GroupKeyDeliverResponse`）
  - 服务端群聊消息、密钥交换通过 JSON `GroupId` 字段路由，不再依赖单一 `client.GroupId`
  - 群聊发送改为 JSON 负载 `{GroupId, Data}` 而非原始字节
- **`ConnectedClient.IsAvailable` 不再受群聊影响**：用户加入群聊后仍可发起/接受私聊
- **客户端去除遗留单群状态**：移除 `_isGroupChat` / `_currentGroupId` / `_currentGroupName` 等单群字段，改用 `_joinedGroupIds` / `_groupNamesById` 多群跟踪
- **`ChatPeer` 清理内部重复定义**：移除与 `Models.cs` 重复的嵌套 `RegisterPayload` 和 `GroupClosedNotice` 记录

### Fixed

- **修复结束聊天后无法再次进入群聊**：
  - 根因：结束聊天移除了本地会话，但服务端仍记录群成员身份，再次双击加入被服务端拒绝
  - 修复：结束聊天仅关闭标签页，保留群成员身份；双击已加入群直接重新打开标签页
- **修复群聊注销权限问题**：服务端对“注销群聊”增加创建者双重校验（`CreatorIdentityId` + `CreatorUsername`）
- **修复群成员退出后人数未实时刷新**：`leave` 后服务端立即广播最新群列表
- **修复在线用户列表可见性问题**：与他人建立会话后对方仍保留在“在线”列表中
- **修复群管理员缺失导致无法注销**：群创建者离开时自动转移管理员；空群首位加入者接任
- **修复只能创建一个群聊的问题**：多群并发支持下创建新群不再强制退出旧群

### Security

- 群聊发送端增加 `JoinedGroups` 校验：服务端对群聊消息、密钥交换请求均验证发送者是否为该群成员
- 群密钥分发透传 `GroupId`，避免跨群密钥混淆

## [1.0.2] - 2026-04-01

### Added

- 群组创建者管理员机制：
  - 创建者身份绑定为群管理员（`CreatorIdentityId`）
  - 新增创建者主动注销群聊能力（`dissolve`）
  - 非创建者注销请求会被拒绝并收到系统提示
- 群组持久化：服务端新增 `groups.json`，支持重启后恢复群组元数据

### Changed

- 客户端 UI 视觉升级（WinForms）：
  - 统一浅色主题与分区背景层级
  - 按钮分级样式（主按钮/危险按钮/中性按钮）
  - 扁平化 + 圆角 + 悬停反馈 + 禁用态统一
  - 输入框、用户列表、状态栏配色优化
  - 会话页签选中态高亮与自绘样式
- 服务端 UI 视觉升级（WinForms）：
  - 与客户端统一的浅色主题与按钮风格
  - 启动/停止/公告按钮统一扁平化、圆角与禁用态反馈
  - 公告输入区、日志区、状态栏配色优化
- 群组生命周期调整：
  - 群组不再因在线人数为 0 自动删除
  - 服务器停止后保留群组元数据，重启可继续加入
  - 群聊注销入口调整到用户列表右键菜单
  - 结束聊天仅执行退出会话，不再承载注销群聊能力

### Fixed

- 修复群聊注销后部分成员仍可在本地继续发言的问题：
  - 服务端注销时向所有在线成员下发 `GroupClosed` 通知并清空其群状态
  - 客户端收到后强制退出该群并清理本地群会话/群密钥

### Notes

- 本版本除 UI 改进外，还包含群组生命周期与群注销流程的协议/状态同步修复。

- 用户名保留规则扩展：新增系统/管理类名称与关键词拦截（客户端 + 服务端双重校验）。

### Changed

  - 移除建联阶段 Safety Number 信任弹窗，改为状态提示

  - 身份卡接收方恢复“是否同意交换”确认

- 用户列表视觉优化：
  - 列表项间距增大
  - 在线用户/群聊/会话增加 emoji 前缀
  - emoji 与文字分离绘制（emoji 使用 `Segoe UI Emoji`，文字使用“微软雅黑”）

### Fixed

- 修复并发写流导致的协议帧交错与断连：客户端与服务端均加入发送串行锁。
- 修复握手竞争导致一端长期“加密建立中”问题：
  - 双向同时发起时按角色收敛
  - 响应方建链后立即触发已建立事件
- 修复会话计数器并发与残留状态问题：

  - 对端会话清理时重置计数与临时握手状态
- 修复 AES-GCM 认证标签不匹配根因：`SignedPreKey` 统一为 X25519 密钥材料（身份签名仍由 Ed25519 完成）。
- 修复正常主动断开被误记为异常断开日志的问题。
- 修复用户列表 emoji 显示为方框/`[]` 问题（含 Header 与群聊项前缀渲染）。

### Security

- 身份卡交换拒绝路径完善：拒绝后状态栏与会话系统消息同步提示。
- 屏蔽用户后，本地拦截其私聊消息与身份卡交换请求。
- 用户名风控增强：保留词精确匹配 + 关键词包含匹配，阻断伪装系统/管理角色名称。

## [0.1.1] - 2026-03-31

### Added

- 跨端身份密钥基础设施（Ed25519 / X25519）
- 注册身份包字段：IdentityPublicKey、SignedPreKeyPublicKey、SignedPreKeySignature
- 私聊签名化握手消息：`MsgX3DHInit (0x30)`、`MsgX3DHResponse (0x31)`
- 私聊消息安全信封：`SecureChatMessage(counter, nonce, tag, ciphertext)`
- 重放防护（按会话 counter 单调递增校验）
- Safety Number 事件与二次确认（Windows/Android）
- 群成员变更签名消息：`MsgGroupMemberSignedChange (0x32)`
- Windows 协议安全自检入口（“自检”按钮）与自检模块

### Changed

- 私聊握手主流程升级为签名化 X3DH 风格
- 群聊 create/join/leave 改为签名变更并服务端验签后执行
- 用户名重复策略统一为：提示修改新用户名（不再接管旧设备）
- Android 本地安全存储改用私有 SharedPreferences
- 左侧列表支持折叠、空占位与会话选中同步
- 聊天消息左右对齐（我在右，对方在左）

### Removed

- 旧私聊未认证密钥交换主路径（legacy MsgKeyExchange）
- 客户端“踢掉旧设备接管”交互流程

### Security

- 引入签名预密钥校验与对端身份包验证
- 私聊消息重放检测与拒绝
- 群成员变更签名校验（服务端强制）

### Build

- 解决方案构建通过：
  - `UnMessage`
  - `UnMessage.Android`
  - `UnMessage.Server`
